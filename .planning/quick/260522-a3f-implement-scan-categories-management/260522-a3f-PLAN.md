# Quick Task: Implement Scan Categories Management section on the Organizer Detail page

This quick task adds a user interface for organizers to manage scan categories dynamically, preventing deletion if records exist, matching the Problem Statements styling, and updating the UI via AJAX without full page reloads.

## Verification Criteria
- [x] Django backend system check passes.
- [x] Organizer can view a list of scan categories on the Hackathon Detail page with scan counts and active status.
- [x] Organizer can create a scan category. The UI updates dynamically.
- [x] Organizer can toggle the active status of a category. The UI updates dynamically.
- [x] Organizer can delete a category if it has no scan records.
- [x] Organizer cannot delete a category if it has scan records; instead, an error notification is shown.
- [x] Category list is ordered by `display_order` then `created_at`.
- [x] Action authorization restricts CRUD to organizers/coordinators only.

## must_haves
- Fetch and annotate `scan_categories` with `scan_count` in `views.py`.
- Prevent deleting a `ScanCategory` in `api_views.py` if `scan_records` exist.
- Build styled UI matching "Problem Statements" section.
- Implement create, toggle status, and delete via REST API using `fetchWithAuth`.
