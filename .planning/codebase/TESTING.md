# Testing

## Frameworks
- The codebase relies on Django's built-in testing framework (`django.test`).
- No external testing frameworks (like `pytest` or `pytest-django`) are explicitly installed in `requirements.txt`.

## Current State
- Test coverage appears minimal based on the dependency tree; development focuses on rapid prototyping and UAT verification.
- Any existing tests would be located in the `tests.py` or `tests/` module within individual Django apps.
