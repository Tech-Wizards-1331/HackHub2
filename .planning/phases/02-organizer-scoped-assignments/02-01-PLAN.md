# Phase 2: Organizer Scoped Assignments (Refinement)

## Overview
This phase adds the management interface for organizers to list and remove coordinators, and to assign them specific responsibilities. This completes the management loop for scoped coordinator assignments.

## Wave 1: Backend Refinement

### 2-01-01: Update Organizer Serializers
- **File**: `backend/organizer/api_serializers.py`
- **Action**: Add `responsibilities` to `HackathonCoordinatorSerializer`.

### 2-01-02: Add Coordinator Management Actions to HackathonViewSet
- **File**: `backend/organizer/api_views.py`
- **Action**:
    - Implement `list_coordinators(self, request, pk=None)` action.
    - Implement `unassign_coordinator(self, request, pk=None)` action (accepts `coordinator_id` or `email`).

## Wave 2: Frontend Implementation

### 2-01-03: Update Organizer Template
- **File**: `frontend/templates/organizer/home.html`
- **Action**:
    - Add a container for the coordinator list in the "Coordinators" tab.
    - Add responsibility checkboxes to the assignment form.
    - Add styles for the coordinator list items.

### 2-01-04: Update Organizer Javascript
- **File**: `frontend/static/js/organizer_dashboard.js`
- **Action**:
    - Implement `fetchCoordinators(hackathonId)` to populate the list.
    - Trigger `fetchCoordinators` when the tab is shown.
    - Update the assignment form handler to include responsibility checkboxes.
    - Implement `unassignCoordinator(hackathonId, coordinatorId)` logic.
